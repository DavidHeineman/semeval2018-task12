Evaluation criteria

Systems participating in the task must classify all instances in the test data set.

The classification results for must be submitted in a delimited text file named answer.txt. Each line consists of two fields separated by horizontal whitespace (a single tab or space character). The first field is the instance ID. The second field is either 0 (which means that warrant0 is the predicted answer) or 1 (which means that warrant1 is the predicted answer).

Sample data, result files, and scorer are available on GitHub.

To submit the results, place answer.txt in a ZIP file (in the top-level directory), and then upload it to CodaLab according to the instructions at Participating in a Competition.

Systems will be scored using accuracy ( accuracy = correct predictions / all instances).
