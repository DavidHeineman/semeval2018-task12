There are two phases of the task: the trial phase and the test phase.

For the trial phase, two datasets are available: training data and dev data.

Both training data and dev data are fully labeled (all instances have an associated gold label). So while it is possible to use both datasets for training your system, we suggest to hold out the dev data for evaluation and error analysis. The scorer for the trial phase is using the dev data as well.

For the test phase, three datasets will be available: training data, dev data (the same as above) and test data.

Test data will be released first when the test phase begins. The data also contains no gold labels, so your system has to make predictions on it.

Notice: the split train/dev/test is exactly the same as in the paper.

Training and development data are available at GitHub

https://github.com/UKPLab/argument-reasoning-comprehension-task/tree/master/mturk/annotation-task/data/exported-SemEval2018-train-dev-test

  • train-full.txt Training data (including gold labels)
  • train-w-swap-full.txt Training data doubled with swapped ordering (including gold labels)
  • dev-only-labels.txt Evaluation data containing only gold for the trial phase (until January 8th, 2018), same as "dev" set in the paper. Against this data are all systems evaluated in the trial phase, so by uploading this file in the submission you can easily get 100% accuracy on the leaderboard :)
  • dev-only-data.txt
    • Evaluation data containing only arguments without gold labels. During the trial phase, you should test your models on this dataset and submit the results. Again, the IDs are the same as the dev set in the paper.
  • dev-full.txt Combines the two above, useful for manual analysis.
